Node.js

When working on Node.js projects, you may encounter the error that start with <--- Last few GCs ---> and then followed by “FATAL ERROR: CALL_AND_RETRY_LAST Allocation failed - JavaScript heap out of memory.” This is a sign that the Node.js process is running out of memory. The default memory limit is 1.5GB, which is not enough for some projects.

To increase the memory limit, you can use the --max-old-space-size flag. One way to set this flag is to add it to the NODE_OPTIONS environment variable:

export NODE_OPTIONS="--max-old-space-size=4096"
